jQuery总结

jQuery是什么？

*jQuery 并不是一种新的语言，是为了简化JavaScript开发而预先封装好了对象和方法的Javascript类库，帮助开发人员建立有高难度交互的Web 2.0特性的富客户端页面，并且兼容各大浏览器。

富客户端与瘦客户端：

  • 富客户端：（Rich Internet Applications，RIA）利用具有很强交互性的富客户端技术来为用户提供一个更高和更全方位的网络体验。
    • 瘦客户端：（Thin Client）指的是在客户端-服务器网络体系中的一个基本无需应用程序的计算机终端。

jQuery的美元符号有什么作用？jQuery库中的有什么作用？jQuery库中的() 是什么?

*$() 是 jQuery库中的工厂函数，其作用：

  1. 可以通过选择器获取页面元素；

  2. 创建页面元素

  3. 将DOM对象转换为jQuery对象

$(document).ready() 是个什么函数？为什么要用它？window.onload 事件和 jQuery ready 函数有何不同？

*window.onload这样的写法作用是希望在页面加载完，自动执行定义js代码（function）。

*$(document).ready(function(){…. })这个函数是用来取代页面中的window.onload

*Onload()和$(document).ready()的区别有以下两点：

1.我们可以在页面中使用多个document.ready()，但只能使用一次onload()；

2.document.ready()函数在页面DOM元素加载完以后就会被调用，而onload()函数则要在所有的关联资源（包括图像、音频）加载完毕后才会调用。

jQuery中有哪几种类型的选择器？

从我自己的角度来讲，可以有4种类型的选择器，如下：

  1. 基本选择器(5)：有id选择器（#）,class选择器（.）,元素名选择器(element),复合选择器(“,”隔开)，通用选择器（*）;
  2. 层次选择器(4)：后代选择器（空格隔开），子代选择器（>）,下一个兄弟（+），下面所以匹配的兄弟（~）；
  3. 表单选择器（11）：匹配表单相关控件的选择器；
  4. 过滤选择器（31）：基本过滤选择器（10），内容过滤选择器（4），可见性过滤选择器（2），属性过滤选择器（7），子元素过滤选择器（4），表单对象属性过滤选择器（4）；

    $(this) 和 this 关键字在 jQuery 中有何不同？

*$(this) 返回一个 jQuery 对象，你可以对它调用多个 jQuery 方法；

*this 代表当前元素，它是 JavaScript 关键词中的一个，表示上下文中的当前 DOM 元素。你不能对它调用 jQuery 方法，直到它被 $() 函数包裹，例如 $(this)

原生js总结

1.立即执行函数

函数的三种定义方式：

函数声明：function fnName () {…};使用function关键字声明一个函数，再指定一个函数名，叫函数声明，只有这种方法定义的函数才会声明提前。

函数表达式：var fnName = function () {…};使用function关键字声明一个函数，但未给函数命名，最后将匿名函数赋予一个变量，叫函数表达式。

匿名函数：function () {}; 使用function关键字声明一个函数，但未给函数命名，所以叫匿名函数，匿名函数属于函数表达式，匿名函数有很多作用，赋予一个变量则创建函数，赋予一个事件则成为事件处理程序或创建闭包等等。

立即执行函数是什么？

Javascript引擎在解析javascript代码时会‘函数声明提前’（Function declaration Hoisting）当前执行环境（作用域）上的函数声明，不会马上执行被提前的函数，函数表达式必须等到Javascirtp引擎执行到它所在行时，才会从上而下一行一行地解析函数表达式，所以如果希望函数被马上执行，就需要特定的写法，JS引擎才会立即执行这个函数。

常见立即执行函数的写法：

(function(){…})()和(function(){…}())；

2.闭包

使用闭包主要是为了设计私有的方法和变量。闭包的优点是可以避免全局变量的污染，缺点是闭包会常驻内存，会增大内存使用量，使用不当很容易造成内存泄露。

闭包的三个特性：

1.函数嵌套函数
2.函数内部可以引用外部的参数和变量
3.参数和变量不会被垃圾回收机制回收

参考：https://segmentfault.com/a/1190000000652891

3.原型和原型链

什么是原型？

在 JavaScript 中，每个对象（null除外）都和另一个对象相关联，这“另一个”对象也被称作原型。

如何访问原型？

每个对象都有__proto__属性，可以通过访问这个属性来确定对象的原型对象。但是__proto__有兼容性问题，建议使用Object.getPrototypeOf()函数，代替__proto__属性查看对象原型。

我们来看一个例子：

var myArray = [1, 2];
myArray.push(3);
myArray.pop();

myAarray这个是我们自己声明的数组对象，但是push方法和pop方法并不是我们写在myArray这个数组对象里面的，为什么我们可以使用呢？

这是因为存在原型链。

什么是原型链？

当我们告诉 JavaScript 我们要调用一个对象的push方法，或读取对象的属性时，运行时会首先查找对象本身。如果运行时找不到想要的东西，它就会循着 __proto__引用和对象原型寻找该成员。当我们调用myArray的push方法时，JavaScript 并没有在myArray对象上发现push方法，而是在myArray的原型对象上找到了，于是 JavaScript 调用此方法。这个找的路径就是对象的原型链。当 JavaScript 查找像 push 方法这样的成员时，它会循着原型引用链检查每一个对象，直到找到该成员，或者抵达原型链的末端，如果找不到就返回undefined。

参考：JavaScript中的原型和继承

4.作用域和作用域链

什么是JavaScript 作用域？

作用域就是变量与函数的可访问范围。在JavaScript中，变量的作用域有全局作用域和局部作用域两种。

全局作用域(Global Scope)

在代码中任何地方都能访问到的对象拥有全局作用域，一般来说以下 3 种情形拥有全局作用域。

1.最外层函数和在最外层函数外面定义的变量拥有全局作用域
2.所有末定义直接赋值的变量自动声明为拥有全局作用域
3.所有window对象的属性拥有全局作用域 
  window对象的内置属性都拥有全局作用域，例如window.name、window.location、window.top等。

局部作用域(Local Scope)

和全局作用域相反，局部作用域一般只在固定的代码片段内可访问到，最常见的例如函数内部，所有在一些地方也会看到有人把这种作用域称为函数作用域 。

作用域链(Scope Chain)

在 JavaScript 中，函数也是对象，实际上，JavaScript 里一切都是对象。函数对象和其它对象一样，拥有可以通过代码访问的属性和一系列仅供 JavaScript 引擎访问的内部属性。其中一个内部属性是 [[Scope]]，由 ECMA-262 标准第三版定义，该内部属性包含了函数被创建的作用域中对象的集合，这个集合被称为函数的作用域链，它决定了哪些数据能被函数访问。

1.在函数创建时，它的作用域链中会填入一个全局对象，该全局对象包含了所有全局变量。

2.函数执行时会创建一个称为“运行期上下文(execution context)”的内部对象，运行期上下文定义了函数执行时的环境。每个运行期上下文都有自己的作用域链，用于标识符解析，当运行期上下文被创建时，而它的作用域链初始化为当前运行函数的[[Scope]]所包含的对象。

3.这些值按照它们出现在函数中的顺序被复制到运行期上下文的作用域链中。它们共同组成了一个新的对象，叫“活动对象(activation object)”，该对象包含了函数的所有局部变量、命名参数、参数集合以及this，然后此对象会被推入作用域链的前端。

4.当运行期上下文被销毁，活动对象也随之销毁。

在函数执行过程中，每遇到一个变量，都会经历一次标识符解析过程以决定从哪里获取和存储数据。该过程从作用域链头部，也就是从活动对象开始搜索，查找同名的标识符，如果找到了就使用这个标识符对应的变量，如果没找到继续搜索作用域链中的下一个对象，如果搜索完所有对象都未找到，则认为该标识符未定义。函数执行过程中，每个标识符都要经历这样的搜索过程

看一个例子：

var scope="global";  
function foo(){  
    console.log(scope);  
    var scope="local"  
    console.log(scope);  
}  
foo();  
//undefined 
//local

这里输出的是undefined和local，为什么不是按照作用域链引用全局作用域的global呢？ 其实是因为在函数作用域内也会有声明提前到函数体的顶部，值保留在原地，所以导致第一个输出是undefined。

5.事件捕获和事件冒泡的区别

事件冒泡和事件捕获实际上是定义具有父子关系的元素上注册事件调用的先后顺序，事件冒泡是子元素事件逐级触发到父元素，事件捕获是父元素有捕获其子元素事件发生的能力，并在这个事件发生之前作出反应

6.什么是同源策略

同domain（或ip）,同端口，同协议视为同一个域，一个域内的脚本仅仅具有本域内的权限，可以理解为本域脚本只能读写本域内的资源，而无法访问其它域的资源。这种安全限制称为同源策略。

他选了她

总该写点东西来纪念我们这两个星期的爱情吧，不知道这能不能称得上叫爱情呢？这段关系很像玫瑰给人的感觉，美丽，但是带刺。

一瓶墨水引起的情愫

认识H先生是缘分，其实认识每一个人都是缘分。当初如果不是因为我和他都同时做了要去学习的决定，那我们也不会认识，如果他没有因为受不了课室前排的不知名味道而决定换座位到我身边与我同桌4个月也不会喜欢我，所以这一切都是注定要发生的。

我问H先生，你是什么时候发现自己喜欢我的？他说，在我看到你不敢看我的时候。

是什么时候发现H先生喜欢我的呢，其实是当我叫他帮我丢垃圾和装水，他调戏我说，亲一口就帮你。然后我mua了一下，他说，你这个人，怎么这么轻浮呀，但是脸上露着淡淡的笑意的时候。

我是什么时候喜欢上H先生的呢？是因为一瓶墨水。那几天我突然想写钢笔字，在姐姐那取回了一支钢笔，打算去买墨水。就是那天H先生说L小姐想见我，我说好。

在珠江新城花城汇的zoo coffee 第一次见到L小姐，聊天之间竟然发现我们思想如此相似，L小姐后来跟H先生说如果不是因为他而认识，我们应该可以做好朋友的。确实，我也有这种感觉。

那天聊了好久，也一起吃了一顿饭，饭桌弥漫着一股尴尬的味道，我本来也没觉得什么，毕竟我不理亏，那个时候我也没有喜欢H先生。

第二天一大早我就到广州图书馆学习去了，H先生死活也要过来一起学习，我挡不住他。他一坐下来，从包了拿出了一瓶英雄墨水，我一脸懵逼，内心却跟开花了一样，脸上开始忍不住，笑得跟傻逼一样。我想这可能就是被爱的感觉吧，我什么都没说过，他就做了，他说昨天回去那么晚了，估计我肯定没买到墨水，上次买对洗碗的手套也要在课室附近买。

我以前希望和恋人之间的默契就是，他嘴上说3分，然后做到10分。然而H先生居然把0做到了10，好感瞬间就一发不可收拾。

H先生还是个人肉GPS，这个真的让我迷得不能自我。看地图溜到不行，方向感好到让我嫉妒。寻路变得好愉快，这是第一次，是和他一起。

这两个星期，去了好多个地方，记得金沙洲的江边，我拿着他的烟，他给我拍了一张堕落女孩的照片；在珠江新城江边的游荡，吹着江风，春暖花开；记得他开车去惠州的高速路上的80km/h的安稳，记得一起爬山，边放歌边下山，我们一起听他喜欢的’午后红茶’，我喜欢的’那谁’；记得我们一起在时尚天河吃过我最爱的酸菜鱼，那天我说‘有机会一起吃冰’，但是没有机会了。

给你一个星期，你选一个吧

其实知道自己对H先生有好感是不对的，一向标榜不要成为自己讨厌的人，然而却一直在打脸，但是我就是缺爱啊，所以想着不如就先自私的享受现在吧，H先生也自私地想同时拥有我们两个，可我还是做出了决定。

那天在珠江边，我说，给你一个星期，你选一个吧。这是第三次推开他，前两次推开，他一联系，我又不忍心了。我说，事不过三，如果这次你没选我的话，我不会再跟你联系了，我会删掉你所有的联系方式。

其实这个选择只是个形式，给我们俩划了一个期限，我很明白他与L小姐之间的羁绊有多深。

如果下雨了，就当我们运气不好吧

广州这些天好像想下雨，一出门都说没带伞等下下雨怎么办，我都是回答说，’如果下雨了，就当我们运气不好吧’。

一个星期后，我们见了最后一次，分开后各回各家，到家附近地铁站开始下暴雨。

我说，今天只是运气不好，我们都好运了这么多次了。

我说，是的。
他说，好好把项目做好，赶紧找份好工作，好好加油。[抱抱]

然后，把他微信删掉了。
